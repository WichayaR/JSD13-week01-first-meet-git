# Product Requirements Document (PRD): Thai Culinary Explorer (ระบบสำรวจอาหารไทย)

เอกสารระบุความต้องการผลิตภัณฑ์นี้จัดทำขึ้นเพื่อออกแบบและพัฒนาเว็บแอปพลิเคชันแนะนำอาหารไทยที่มีความสวยงาม ทันสมัย และเป็นระบบอินเตอร์แอคทีฟ โดยมีจุดประสงค์หลักเพื่อการจัดระเบียบข้อมูลและแสดงผลลัพธ์ผ่านการใช้ JavaScript Array และ Object ร่วมกับ HTML5 และ CSS3 (Vanilla) ที่เน้นความพรีเมียม

---

## 1. ข้อมูลภาพรวม (Project Overview)
เว็บแอปพลิเคชัน **"Thai Culinary Explorer" (เสน่ห์สำรับไทย)** เป็นแพลตฟอร์มแบบ Single Page Application ที่รวบรวมอาหารไทยขึ้นชื่อจากภูมิภาคต่าง ๆ มาแสดงผลในรูปแบบการ์ดที่สวยงาม ผู้ใช้สามารถกรองประเภท ค้นหา ค้นพบบสูตรอาหาร และทดลองจำลองสั่งอาหารหรือคำนวณโภชนาการได้แบบเรียลไทม์

### ไฟล์ประกอบโครงการ
1. **[PRD.md](file:///C:/Users/wicha/OneDrive/Desktop/JSD13/PRD.md)**: เอกสารวางแผนและข้อกำหนดระบบ (ไฟล์นี้)
2. **`index.html`**: โครงสร้างหน้าเว็บหลัก ประกอบด้วยส่วนหัว (Hero Section), ส่วนกรองข้อมูล (Filter Panel), ส่วนแสดงผลรายการอาหาร (Grid System) และส่วนแสดงรายละเอียดอาหารเชิงลึก (Detail Modal/Panel)
3. **`style.css`**: จัดการด้านสุนทรียศาสตร์ (Aesthetics) สไตล์หรูหรา โมเดิร์น ใช้โทนสีอุ่นแนวสมุนไพรและเครื่องเทศไทย
4. **`array.js`**: จัดการชุดข้อมูลรายการอาหารทั้งหมดในรูปแบบ Array ของ Objects โดยมีฟังก์ชันการกรอง (Filter), เรียงลำดับ (Sort), ค้นหา (Find), และวิเคราะห์ข้อมูล (Reduce)
5. **`object.js`**: จัดการโครงสร้างข้อมูลของอาหารแต่ละจานในรูปแบบ Nested Object ที่ซับซ้อนขึ้น พร้อมมีฟังก์ชันหรือ Method ในตัววัตถุสำหรับคำนวณเฉพาะเจาะจง

## 2. การออกแบบและสุนทรียศาสตร์ (UX/UI & Aesthetics)
เพื่อให้ผู้ใช้งานรู้สึก "ว้าว" ตั้งแต่แรกเห็น เราจะกำหนดแนวทางการออกแบบดังนี้:

* **Theme Color Palette (Warm Spicy & Herbal Gold):**
  * **Primary (Gold/Warm Amber):** `#D4AF37` / `#C5A059` (สีทองแห่งความเป็นไทย, สีขมิ้น)
  * **Secondary (Herbal Green):** `#2D5A27` / `#3A7D44` (สีใบตอง, ตะไคร้)
  * **Accent (Chili Red):** `#B22222` / `#D32F2F` (สีพริกแดงกระตุ้นความอยากอาหาร)
  * **Background (Dark Charcoal & Warm Off-White):** `#121212` (สำหรับ Dark Mode พรีเมียม) หรือไล่ระดับสี (Gradient) แบบ Warm Gradient
* **Typography:** ใช้ Google Fonts
  * ภาษาไทยและอังกฤษหลัก: **"Kanit"** หรือ **"Sarabun"** ผสมกับ **"Cinzel"** สำหรับหัวข้อหลัก (เพิ่มความคลาสสิก)
* **Visual Effects:**
  * **Glassmorphism:** ใช้ `backdrop-filter: blur(10px)` และขอบสีขาวโปร่งแสงสำหรับส่วนควบคุมและ Modal
  * **Card Hover:** เมื่อวางเมาส์เหนือการ์ดอาหาร การ์ดจะยกตัวขึ้นเล็กน้อย มีเงาฟุ้ง และแสดงภาพอาหารชัดขึ้นพร้อมสีขอบทอง
  * **Micro-animations:** แอนิเมชันตอนโหลดภาพ และปุ่มกดที่ยุบตัวได้เล็กน้อยเมื่อคลิก

## 3. โครงสร้างข้อมูล JavaScript

### A. ชุดข้อมูลหลักใน `array.js` (Array of Objects)
`array.js` จะบรรจุข้อมูลรายการอาหารหลักอย่างน้อย 6 เมนู ครอบคลุม 4 ภาค:
1. **ต้มยำกุ้ง** (ภาคกลาง)
2. **ส้มตำถาด** (ภาคอีสาน)
3. **น้ำพริกอ่อง** (ภาคเหนือ)
4. **แกงไตปลา** หรือ **คั่วกลิ้ง** (ภาคใต้)
5. **ผัดไทยกุ้งสด** (ภาคกลาง - อาหารสตรีทฟู้ดระดับโลก)
6. **ข้าวซอยไก่** (ภาคเหนือ)

#### ตัวอย่างโครงสร้างข้อมูล Array:
```javascript
const thaiDishes = [
  {
    id: 1,
    nameTH: "ต้มยำกุ้ง",
    nameEN: "Tom Yum Goong",
    region: "Central",
    spiciness: 4, // ระดับ 1-5
    calories: 150,
    price: 250,
    imageUrl: "images/tom_yum.jpg", // จะเจนจริงด้วย AI
    tags: ["Spicy", "Soup", "Seafood"]
  },
  // ... รายการอื่นๆ
];
```

#### ฟังก์ชันที่ต้องพัฒนาใน `array.js`:
* `filterByRegion(region)`: กรองอาหารตามภาคที่เลือก
* `sortByPrice(order)`: เรียงราคาจากน้อยไปมาก หรือมากไปน้อย
* `sortBySpiciness()`: เรียงตามระดับความเผ็ดร้อน
* `calculateAverageCalories()`: คำนวณค่าเฉลี่ยแคลอรีของรายการทั้งหมดโดยใช้ `Array.prototype.reduce()`
* `searchDishByName(keyword)`: ค้นหาอาหารโดยใช้ชื่อภาษาไทยหรืออังกฤษ

### B. โครงสร้างรายละเอียดใน `object.js` (Nested Objects & Methods)
`object.js` จะแสดงรายละเอียดขั้นสูงของอาหารจานที่ผู้ใช้เลือก (Selected Dish Details) และทำงานกับโครงสร้างออบเจกต์ที่มีความลึก (Nested Object)

#### ตัวอย่างโครงสร้างข้อมูล Object:
```javascript
const selectedDishDetail = {
  id: 1,
  nameTH: "ต้มยำกุ้ง",
  description: "ต้มยำน้ำข้นรสชาติกลมกล่อม เผ็ด เปรี้ยว เค็ม หวาน หอมกลิ่นสมุนไพร ข่า ตะไคร้ ใบมะกรูด",
  ingredients: [
    { name: "กุ้งแม่น้ำ", amount: 200, unit: "กรัม" },
    { name: "เห็ดฟาง", amount: 50, unit: "กรัม" },
    { name: "ข่า", amount: 15, unit: "กรัม" },
    { name: "ตะไคร้", amount: 20, unit: "กรัม" },
    { name: "ใบมะกรูด", amount: 5, unit: "กรัม" }
  ],
  nutrition: {
    protein: "15g",
    fat: "5g",
    carbs: "8g",
    sodium: "980mg"
  },
  recipeSteps: [
    "ต้มน้ำซุป/หางกะทิให้เดือด ใส่ข่า ตะไคร้ ใบมะกรูดลงไปต้มจนส่งกลิ่นหอม",
    "ใส่เห็ดฟางและกุ้งแม่น้ำ ต้มจนกุ้งเริ่มสุก",
    "ใส่น้ำพริกเผา นมข้นจืด และพริกขี้หนู",
    "ปิดไฟ แล้วปรุงรสด้วยน้ำปลา และน้ำมะนาวตามชอบ ตกแต่งด้วยผักชี"
  ],
  // Method ใน Object
  calculateIngredientScale: function(servings) {
    return this.ingredients.map(item => ({
      name: item.name,
      amount: item.amount * servings,
      unit: item.unit
    }));
  }
};
```

## 4. ฟังก์ชันหลักบนหน้าเว็บ (Key Web Features)

1. **Header & Hero Banner:**
   * สโลแกนเท่ ๆ พร้อมภาพพื้นหลังอาหารไทยพรีเมียม
   * แถบค้นหาค้นหาอาหารแบบ Real-time Typing

2. **Filter & Control Hub (Sidebar หรือ แถบควบคุมด้านบน):**
   * ปุ่มสำหรับเลือกฟิลเตอร์ภูมิภาค (ทั้งหมด, เหนือ, กลาง, อีสาน, ใต้)
   * ปุ่มเรียงลำดับราคาและระดับความเผ็ด
   * ปุ่มแสดงสถิติ (Statistic Panel): คลิกเพื่อคำนวณแคลอรีเฉลี่ยในระบบ หรือราคาเฉลี่ย โดยเรียกใช้ฟังก์ชันจาก `array.js`

3. **Dish Grid Gallery:**
   * การ์ดอาหารแสดงรูปภาพ, ชื่อสองภาษา, แถบไอคอนแสดงระดับความเผ็ด (เช่น ไอคอนรูปพริก 1-5 เม็ด)
   * แสดงป้าย Tag (เช่น แกง, ยำ, ทอด)
   * เมื่อกดคลิกการ์ด จะทำการดึงข้อมูลเชิงลึกมาแสดงใน **Detail Panel**

4. **Culinary Detail Viewer (Modal หรือ Side Panel):**
   * เมื่อคลิกการ์ด ระบบจะโหลดออบเจกต์อาหารนั้น ๆ จาก `object.js` มาเรนเดอร์ในสไตล์ที่สวยงาม
   * แสดงวัตถุดิบและสัดส่วน พร้อมแถบปรับจำนวนที่รับประทาน (Servings Slider/Selector) เช่น 1 คน, 2 คน, 4 คน เมื่อปรับค่าจะเรียกใช้ Method `calculateIngredientScale(servings)` แล้วอัปเดตตัวเลขวัตถุดิทันที
   * แสดงข้อมูลโภชนาการแบบกราฟหรือการ์ดเล็ก ๆ
   * แสดงขั้นตอนการปรุงอาหารทีละสเต็ป (Step-by-step cooking guide)
